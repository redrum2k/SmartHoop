# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Setup

```bash
pip install -r requirements.txt
```

Requires Python 3.10+ and `yolov8n.pt` (auto-downloaded by ultralytics on first run).

## Running

```bash
# Calibrate zones (required once per camera position)
python main.py --calibrate --source 0

# Run with webcam
python main.py --source 0

# Run on video file
python main.py --source inputs/testvid_1.mov

# Use HSV red segmentation instead of YOLO (debugging)
python main.py --hsv --source 0
```

There are no automated tests. Manual testing is done by running on `inputs/testvid_1.mov` and verifying the hit/backboard/miss counters match expected counts.

## Architecture

The pipeline is: **detect → track → classify**, executed per-frame in `main.py:run_detection_loop`.

### Data flow

1. `BallDetector.detect(frame)` → `List[BallDetection]`  
   Uses YOLOv8 (class 32 = sports ball) or HSV red segmentation fallback. Returns bounding box, center, confidence.

2. `BallTracker.update(detections, dt)` → `Optional[TrackedBall]`  
   Nearest-neighbor association against last known position. Maintains a fixed-length deque of `(cx, cy)` history. During occlusion (no detections), extrapolates position using last velocity (or Kalman if `use_kalman: true`). Resets after `max_occlusion_frames` consecutive misses.

3. `EventClassifier.update(tracked_ball, fps)` → `ClassifierResult`  
   State machine: `IDLE → ATTEMPT → CLASSIFIED → IDLE`. Transitions:
   - `IDLE → ATTEMPT`: ball enters `attempt_roi` (or always, if no ROI configured)
   - Hit: trajectory crosses rim downward (`ball_crosses_rim_downward`) with `vy >= downward_velocity_threshold`, confirmed for `K_hit_frames` consecutive frames
   - Backboard: bbox overlaps `backboard_zone` for `K_backboard_frames` frames
   - Miss: ball exits ROI or `T_end_sec` elapses with no other outcome
   - `CLASSIFIED → IDLE`: after `cooldown_sec`

### Zone geometry (`vision/zones.py`)

Zones are either `ZonePolygon` (list of points) or `ZoneEllipse` (center, axes, angle). Key helpers:
- `point_in_zone` — point-in-polygon (OpenCV) or ellipse math
- `bbox_overlaps_zone` — checks center + 4 corners against polygon
- `ball_crosses_rim_downward` — iterates trajectory history, checks above→below crossing inside hoop zone

### Configuration

All tuning parameters live in `config.yaml`. Key sections: `detection`, `tracking`, `classification`, `performance`. The `zones.json` file stores calibrated polygon coordinates and is generated by `--calibrate` mode.

### Output

Each classified shot prints a JSON log line to stdout:
```json
{"event": "hit", "timestamp": "...", "attempt_id": 1, "confidence": 0.92, "notes": "rim-crossing"}
```
